All About GitHub

  • Q. What is a version control system?
  • A. Tracks changes to source code.
  • Q. What is Git?
  • A. Version control software.
  • Q. What is GitHub?
  • A. Platform for hosting, managing, and sharing code. Runs on Git.
  • Q. What is the difference between Git and GitHub?
  • A. Git is backend software. GitHub is a file-hosting service based on Git functionality.
  • Q. Who started Github and how was it started?
  • A. Founded by Tom Preston-Werner, Chris Wanstrath, P. J. Hyett, and Scott Chacon, using Ruby on Rails.
  • Q. What company owns it now?
  • A. Microsoft.
  • Q. How much does a GitHub account cost?
  • A. The basic plan is free, the team plan is $4 per month, and the enterprise plan is $21 per month.

Git/GitHub terms:

  • repository: Collection of files. Similar to a desktop folder.
  • commit: Basically the save button for a repository. Can contain a message for context.
  • fork: Clone of another repository at a specific moment in time. Used to take that repository in a different (forked) direction than the original in some way.
  • push: Basically the publish button. To push is to send changes from your local repository (i.e. your computer) to a remote repository (i.e. GitHub).
  • pull requests: Basically the download button. Updates your local repository with new commits from a remote repository.
  • workflows: Automated processes that will run jobs for you based on triggered events.
  • issues: Communication and documentation feature. For tracking bug reports, ideas, plans, or progress on a project with a team.
  • raw button: For viewing the actual text contents of a file, e.g., HTML tags will be visible in the web page instead of applying to its graphical presentation.
  • blame button: For getting timestamped information on who last committed what, and where, line by line in a file.
