Computer Architecture Terms
1. RAM
Random access memory. The technique computers use for accessing any data stored in a memory cell at the same speed as any other cell.
2. cache
Where data nearby what was intentionally accessed gets copied to and stored. Much smaller but faster than RAM.
3. Principle of locality: The high likelihood that when a program fetches a piece of data or an instruction:
1. "It will access that same instruction or piece of data in the very near future."
2. "It will soon access the instructions or data that are located near that piece of data, where 'near' means an address whose numerical value is close to this one."
4. ROM
Read-only memory. A "special type of random access memory into which information has been prerecorded during manufacture." For booting up a computer.
5. memory cell size
Number of bits per cell. Nowadays, standard is 8, a byte.
6. memory cell address
Location of each cell, not the contents.
7. maximum memory size
Maximum theoretical amount of memory cells a computer can be equipped with, not just its current size of memory.
8. register
Special high-speed storage cell for operands and holding results of completed math operations. Similar to RAM cells, but without addresses or general-purpose storage.
9. fetch
Get me a copy of the data from that address.
10. store
Overwrite the data in that address with this data.
11. memory access time
How long it takes to fetch or store data. Same between all addresses.
12. DASD
Direct access storage device(s). "[E]very unit of information still has a unique address, but the time needed to access that information depends on its physical location and the current state of the device." Hard drives, DVDs.
13. Arithmetic and Logic Unit (ALU)
Subsystem for math and logical operations: addition, subtraction, equality checks.
14. Control Unit
Subsystem for fetching data, decoding what to do with that data, and executing appropriate commands on that data with help from the ALU or memory. These days, the ALU and control unit are both just part of the processor.

Four things the interface between the computer and the hardware does:
1. "Hides from the user the messy details of the underlying hardware."
2. "Presents information about what is happening in a way that does not require in-depth knowledge of the internal structure of the system."
3. "Allows easy user access to the resources available on this computer."
4. "Prevents accidental or intentional damage to hardware, programs, and data."

Responsibilities of systems software:
1. "Hides the complex and unimportant (to the user) details of the internal structure of the Von Neumann architecture."
2. "Presents important information to the user in a way that is easy to understand."
3. "Allows the user to access machine resources in a simple and efficient way."
4. "Provides a secure and safe environment in which to operate."

Terms
1. systems software
Collection of programs managing and facilitating access to resources.
2. user interface
Nowadays, the GUI. Text-based or graphical ways the user can control the computer.
3. language services
Programs for programming. Assemblers, compilers, interpreters.
4. memory managers
Programs for allocating and reclaiming memory space.
5. information managers
Programs for handling data on storage devices like flash drives.
6. i/o managers
Software packages for using different kinds of I/O devices.
7. scheduler
Program for selecting what programs to run. Lets user run multiple programs at once.
8. utilities
Library routines collections. "Text editors, online help routines, image and sound applications, and control panels are examples of utility routines."
9. assembly language
Low-level programming language. Developed to let people escape the soup of binary machine language.

Functions of an Operating System
1. "User interface management (a receptionist)"
2. "Control of access to the system and to data files (a security guard)"
3. "Program scheduling and activation (a dispatcher)"
4. "Efficient resource allocation (an efficiency expert)"
5. "Deadlock detection and error detection (a traffic officer)"
